# Lab: VGA Boot Messages (启动信息打印)

[English](../../README.md)

难度: ★☆☆☆☆

## 设计初衷

xv6 从 BIOS 跳入 bootasm.S 到 bootmain.c 加载内核，再到 entry.S 跳转 main()，整个启动过程屏幕一片漆黑——没有任何视觉反馈。直到 console 初始化完成，用户才看到 `init: starting sh`。

真实 PC 的启动过程是有反馈的：BIOS 自检信息、GRUB 的 Loading 字样、Linux 的 penguin logo。本实验在 bootmain.c 中直接写 VGA 文本缓冲区，让启动过程的每个阶段都可见。

核心问题：*"在内核还没跑起来之前，怎么在屏幕上打字？"*

## 前置知识

- **VGA 文本模式**: PC 启动后默认进入 80x25 文本模式，显存映射在物理地址 `0xB8000`。每个字符占 2 字节：低字节是 ASCII 码，高字节是颜色属性
- **实模式/保护模式**: bootasm.S 已经切换到保护模式，但 0xB8000 的物理地址仍然可以直接访问（bootloader 阶段没有分页）
- **颜色属性字节**: `高4位=背景色 | 低4位=前景色`，如 `0x07`=灰底白字, `0x02`=黑底绿字, `0x0C`=黑底亮红字

```
VGA 颜色表:
0=黑 1=蓝 2=绿 3=青 4=红 5=紫 6=棕 7=灰
8=深灰 9=亮蓝 A=亮绿 B=亮青 C=亮红 D=亮紫 E=黄 F=白
```

## 实验内容

### 1. 实现 VGA 输出函数 (boot/bootmain.c)

在 bootmain.c 中实现两个基础函数：

- `vgaputc(char c, uchar color)` — 在光标位置写入一个字符，推进光标。需要处理 `\n` 换行和屏幕滚动（cursor 超过 80x25 时将内容上移一行）
- `vgaprint(char *s, uchar color)` — 遍历字符串逐个调用 vgaputc

**关键接口**：VGA 文本缓冲区起始地址 `0xB8000`，类型为 `ushort*`，每个元素的低字节是字符，高字节是颜色属性。

**提示方向**：
- 用一个静态变量 `cursor` 跟踪当前位置（row * 80 + col）
- 滚屏：将前 24 行复制到前 24 行，第 25 行清空为空格

### 2. 在 bootmain() 各阶段插入打印

在 `bootmain()` 的关键节点调用 vgaprint，用不同颜色区分阶段：

| 位置 | 建议颜色 | 内容 |
|------|---------|------|
| 函数入口 | 0x07 (灰白) | bootloader 标题 |
| ELF magic 校验前 | 0x02 (绿) | 磁盘读取状态 |
| ELF 校验失败 | 0x0C (红) | 错误信息 |
| 段加载循环 | 0x0B (亮青) | 解析状态 |
| 跳转 entry 前 | 0x0E (黄) | 控制转移提示 |

### 3. 约束：512 字节限制

bootmain.c + bootasm.S 必须 ≤ 510 字节（bootloader 硬限制）。添加的字符串和函数会增加代码体积。用 `ls -l build/bootblock` 检查大小。如果超限，缩短字符串或用缩写。

## 涉及的 OS 概念

| 概念 | 在本实验中的体现 |
|------|----------------|
| Memory-Mapped I/O | VGA 缓冲区不是"写端口"，而是直接写内存地址 0xB8000 |
| 实模式 vs 保护模式 | bootasm.S 已切换保护模式，但 VGA 地址仍可直接访问 |
| Bootloader 空间限制 | 512 字节扇区 → 实际可用 510 字节 + 0x55AA 签名 |
| 线性地址空间 | 保护模式下没有段偏移，地址就是地址 |

## 验证

### 编译和运行

```bash
make clean && make qemu
```

### 验证目标

| 目标 | 预期行为 | 观察方式 |
|------|---------|---------|
| VGA 输出可见 | 启动时显示彩色文字 | QEMU 窗口启动瞬间观察 |
| 颜色区分 | 不同阶段不同颜色 | 灰色标题、绿色 I/O、红色错误 |
| bootblock 大小 | ≤ 510 字节 | `ls -l build/bootblock` |
| console 覆盖 | VGA 输出在 console 初始化后被覆盖 | 正常现象 |

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 编译报 sign.pl 错误 | bootblock 超过 510 字节 | 缩短字符串，减少函数 |
| 花屏/乱码 | cursor 初始值不对或溢出 | 检查 cursor 范围 [0, 80*25) |
| 看不到输出 | console 初始化覆盖了内容 | 正常——可改 console.c 延迟清屏 |
| 颜色不对 | 属性字节高低位写反 | 检查 `(color << 8) \| char` 的顺序 |

## 关键代码路径

- VGA 缓冲区写入: boot/bootmain.c → 直接写 `0xB8000 + cursor`
- 滚屏处理: boot/bootmain.c → 行复制 + 末行清空
- 各阶段打印: boot/bootmain.c → vgaprint 插入在 readseg/ELF 解析前后

## 设计权衡

| 方面 | 无启动输出 (原始) | VGA 打印 (本实验) |
|------|----------------|-----------------|
| 用户体验 | 启动过程完全黑暗 | 每阶段有视觉反馈 |
| 代码空间 | 510 字节全部可用 | 需要给 VGA 函数和字符串留空间 |
| 调试难度 | 只能看最终结果 | 可以看到卡在哪个阶段 |
