# Lab: Graphical Boot Screen (图形模式启动画面)

[English](../../README.md)

难度: ★★★★☆

## 设计初衷

xv6 启动后只有文字——80x25 的 VGA 文本模式。但 PC 的显卡支持图形模式，可以画像素。现代操作系统（Windows、macOS）的启动过程都有图形界面和进度动画，这些都是在内核完全启动前绘制的。

本实验在 bootloader 阶段切换到 VBE 图形模式，用帧缓冲区直接画像素，实现一个带进度条的启动画面。

这个实验连接了两个世界：**实模式的 BIOS 中断能力**（设置图形模式）和**保护模式的内存直写**（在帧缓冲区上画图）。

## 前置知识

- **VBE (VESA BIOS Extensions)**: 标准 VGA 只有 320x200x256 色，VBE 扩展到更高分辨率。通过 BIOS `int 0x10` 的 `AX=0x4F02` 设置
- **BIOS 中断**: 只能在实模式（16位）下调用。bootasm.S 在切换保护模式之前可以调用
- **帧缓冲区**: 显卡映射到物理内存的像素数组。典型地址 `0xE0000000` 附近，每个像素 3-4 字节 (RGB)
- **分页映射**: xv6 开启分页后，帧缓冲区物理地址需要映射到虚拟地址才能继续访问

### 执行时序

```
bootasm.S (实模式):  调用 int 0x10 切换图形模式 → 保存帧缓冲区地址 → 切换保护模式
bootmain.c (保护模式, 未分页):  读磁盘 → 在帧缓冲区上绘制进度条 → 跳转 entry.S
entry.S (保护模式, 开分页):  映射帧缓冲区物理地址到虚拟地址
kernel/console.c:  输出函数改为写帧缓冲区
```

## 实验内容

### 1. 在实模式设置 VBE 图形模式 (修改 boot/bootasm.S)

在切换保护模式之前，调用 BIOS `int 0x10` 设置图形模式。

**需要做的事**：
- 调用 `int 0x10, AX=0x4F01` 获取指定 VBE mode 的信息，结果存到 `0x8000`
- 从 mode info 的偏移 `0x28` 处提取帧缓冲区物理地址，保存到约定位置（如 `0x7000`）
- 保存分辨率（XResolution、YResolution）到 `0x7004` 和 `0x7006`
- 调用 `int 0x10, AX=0x4F02` 设置图形模式（BX 中 bit 14 置位表示使用线性帧缓冲区）

**VBE mode number 参考**（因 QEMU 版本而异）：
- `0x143`: 1024x768x32
- `0x118`: 1024x768x24
- `0x112`: 640x480x24（最安全的备选）

### 2. 在 bootmain.c 中绘制启动画面 (修改 boot/bootmain.c)

**需要实现的函数**：
- `fb_init()` — 从约定地址读取帧缓冲区地址和分辨率
- `fb_pixel(x, y, r, g, b)` — 在指定坐标画一个像素（32bpp 模式下每个像素 4 字节）
- `fb_rect(x, y, w, h, r, g, b)` — 画一个填充矩形
- `fb_progress(percent)` — 画进度条（灰色背景 + 蓝色填充）

**关键接口**：帧缓冲区基地址通过 `0x7000` 获取，pitch = width * 4（32bpp），像素地址 = base + y * pitch + x * 4。

### 3. 在 entry.S 中映射帧缓冲区 (修改 kernel/entry.S)

帧缓冲区物理地址（如 `0xE0000000`）在开启分页后无法直接访问。

**两种方案**：
- 在 `entrypgdir` 中添加 4MB 大页映射（简单但不够灵活）
- 在 `kvmalloc()` 完成后用 `mappages()` 建立 4KB 映射（更精确）

### 4. 修改 console.c 支持帧缓冲区 (修改 kernel/console.c)

这是最大的改动——将输出目标从 VGA 文本模式 (0xB8000) 改为帧缓冲区。

**需要实现**：
- 位图字体渲染器：用 8x16 的字体数据，逐像素绘制每个字符
- 文字光标跟踪（cons_col, cons_row）
- 帧缓冲区滚动（内容上移一行）

**提示**：可以使用 PC BIOS 内置字体数据，或自己定义一个精简的 ASCII 字模数组。

## 涉及的 OS 概念

| 概念 | 在本实验中的体现 |
|------|----------------|
| BIOS 中断服务 | `int 0x10` 只在实模式可用 |
| VBE/VESA | 真实显卡驱动的抽象层 |
| 帧缓冲区 (framebuffer) | 显存的线性映射，写像素 = 写内存 |
| 物理地址 vs 虚拟地址 | 帧缓冲区在分页前后映射方式不同 |
| 页表映射 | 4MB 大页 vs 4KB 细粒度映射 |
| 位图字体 | 不依赖 GPU 的纯 CPU 文字渲染 |

|------|---------|------|
| boot/bootasm.S | 修改 | VBE 模式设置（实模式部分） |
| boot/bootmain.c | 修改 | 帧缓冲区绘制函数 |
| kernel/entry.S | 修改 | entrypgdir 映射帧缓冲区 |
| kernel/main.c | 修改 | 传递帧缓冲区信息 |

| 目标 | 预期行为 | 观察方式 |

| 进度条绘制 | 蓝色进度条从左到右 | 观察 QEMU 窗口 |

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| QEMU 显示不变 | VBE mode number 不对 | 尝试 0x112 (640x480) |
| 三重启 | 帧缓冲区地址无效 | 检查 0x7000 处保存的地址 |
| 开分页后花屏 | entrypgdir 未映射帧缓冲区 | 在 entry.S 添加映射 |
| 文字乱码 | 字体数据不正确 | 检查 font_bitmap 数组 |
| 性能很差 | 逐像素绘制太慢 | 考虑一次写一行 |

## 关键代码路径

- VBE 设置: boot/bootasm.S → 实模式 int 0x10 → 地址/分辨率存 0x7000
- 帧缓冲区初始化: boot/bootmain.c → 从 0x7000 读取参数
- 像素绘制: boot/bootmain.c → 直接写帧缓冲区内存
- 页表映射: kernel/entry.S 或 kernel/vm.c → 映射帧缓冲区物理地址
- 文字渲染: kernel/console.c → 位图字体逐像素绘制

## 设计权衡

| 方面 | VGA 文本模式 (原始) | VBE 图形模式 (本实验) |
|------|--------------------|--------------------|
| 分辨率 | 80x25 字符 | 1024x768+ 像素 |
| 颜色 | 16 色 | 24/32 位真彩色 |
| 字体 | 硬件内置 | 需要软件渲染 |
| 复杂度 | 极简 | 需要字体、映射、滚动 |
| 启动速度 | 快 | 慢 (CPU 渲染文字) |
